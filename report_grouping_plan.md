# תכנית יישום: עדכון דו"ח חודשי - חלוקה לפי ספק ומוצרים
**מטרה:** שינוי מבנה הדו"ח החודשי כך שיציג את הנתונים לפי ספקים, ובתוך כל ספק יציג פירוט של המוצרים שהוזמנו, הכמויות והסכומים.

## 1. עדכון מודל הנתונים (Data Processing) ב- `js/monthly-report.js`
*   עדכון הפונקציה `renderSupplierBreakdown` או יצירת פונקציה חדשה שתעבור על כל ההזמנות וכל הפריטים.
*   איסוף נתונים במבנה הבא:
    ```json
    {
      "supplierId": {
        "name": "שם הספק",
        "total": 1500,
        "count": 5,
        "products": {
          "productId": {
            "name": "תפו\"א",
            "quantity": 50,
            "unit": "ק\"ג",
            "total": 250
          }
        }
      }
    }
    ```

## 2. עדכון ממשק המשתמש (UI) ב- `monthly-report.html`
*   שינוי העיצוב של אזור `supplier-breakdown` מכרטיסיות פשוטות לכרטיסיות נפתחות (Accordion).
*   הוספת אפשרות לחיצה על כרטיסיית הספק.
*   בפתיחה, הצגת טבלה קטנה או רשימה של כל הפריטים שהוזמנו מאותו ספק באותו חודש, כולל כמות מרוכזת וסכום מרוכז.

## 3. עדכון ייצוא ל-Excel ב- `js/export.js` (אופציונלי)
*   וידוא שהייצוא תומך או מציג את הנתונים ברמת הפריט ולא רק ברמת ההזמנה, כדי להתאים לתצוגה החדשה (אם נדרש).

## 4. בדיקות
*   יצירת הזמנות דמה למספר ספקים עם מספר פריטים.
*   וידוא שהכמויות מצטברות נכון.
*   וידוא שהסכומים זהים לסך ההוצאות.
